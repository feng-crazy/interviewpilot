📋 Schema Changes Summary

➕ New Table: JobPosition

  • Extracted job-related fields from Interview
  • Fields: jd_text, company_info, interviewer_info, interview_scheme, constraint_info
  • Enables reuse of job definitions across multiple interviews
  • Supports job position management independently

➕ New Field: Interview.resume_text

  • Stores candidate's resume content
  • Enables AI to reference resume during interview
  • Supports personalized question generation

➕ New Foreign Key: Interview.job_position_id

  • References JobPosition.id
  • Establishes 1:N relationship (one job, many interviews)
  • Replaces embedded job fields

➖ Removed Fields from Interview

  • jd_text → moved to JobPosition
  • company_info → moved to JobPosition
  • interviewer_info → moved to JobPosition
  • interview_scheme → moved to JobPosition
  • constraint_info → moved to JobPosition

🔄 Unchanged Tables

  • ChatMessage - no changes
  • InterviewReport - no changes
  • Existing Interview fields preserved

🎯 Benefits

  • Normalized data model
  • Job positions reusable across interviews
  • Clearer separation of concerns
  • Supports future job management features
